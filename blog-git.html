Git

December, 2020

Git is a version control tool. Simple isn't it? No no .. What is this version control? So every day or I would say every hour a programmer writes some code and save it. So what if the code was working a few hours ago and after making some changes it stops working or I don't want the code that I have written recently or I want to backtrack to any last saved code. Saving a code to Git is commit and each commit is like version of your code. So in simple words git controls the versions of the work we have saved to it.

Git tracks the changes by different users in different computers and saves it all in a team(or individually) working together.

    Useful terms in git

  • Commit- saving your work.
  • Repository-container or it's similar to folder.
  • Branch-In a Repository there is a main branch which is analogous to a bucket which stores everything and each member of the tream can create other branches for some purpose which is analogous to mug. Merging of branch to main is analogous to pouring of content of mug (individual developer's branch) to bucket(main).

    • One Important task in Git is creating a branch and merging it. The following steps may help you next time in dealing with the branch.
    1. Create branch- When you create branch it is good practice to give name like this- "developerName-moduleName".

      After creating branch you can work in the module you have created branch for and save it in editor.
    2. Commit- To commit the work in the branch you can go to GitHub Desktop and click on commit and dont forget to give meaningfull commit messages.
    3. publish branch-Publish the current branch in GitHub Desktop and it's a good idea to view the branch in your git profile.
    4. Pull Request- Now you have to make pull request either from GitHub Desktop or your git profile.
    5. Merge Branch- After making pull request you can preview the work/code you have done the branch and also share with your friends for review then click on Merge Branch.
    6. Pull Origin-The last step to click on pull origin in GitHub Desktop and Wooaah You did it.
